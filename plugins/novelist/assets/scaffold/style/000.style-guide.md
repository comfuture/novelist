## Manuscript Markdown

- Only `## Draft` is published. Keep Synopsis and Revision Notes as editorial source.
- Separate plain narration paragraphs with a blank line.
- Wrap each audible speech range as exact `*“…”*`, whether it stands alone or
  shares a paragraph with narration, as in `*“Approved.”* Rhea said.` Curly
  double quotes identify speech, while the outer emphasis maps only that range
  to serif italics in EPUB. Blank lines determine paragraph boundaries.
- Reserve unquoted `*…*` for interior thought. Use `‘…’` for cited wording,
  remembered phrasing, or a quotation nested inside speech; do not use curly
  double quotes for non-spoken mentions.
- Use `**…**` only for genuine strong emphasis. Balance every marker on one line.
- Use inline backticks only for literal machine output, UI labels, filenames, code identifiers, or log text.
- Put `---` alone between blank lines for a scene or time break. Never use `* * *`.
- Do not use lists, blockquotes, or fenced code blocks anywhere inside Draft. Balance inline markers on one line.

Publication styles prose paragraphs with a sans-serif stack and converts each
speech range to `<i class="dialog">` with a serif italic stack. Because reading
software may override fonts, typography is reinforced by the semantic class and
quotation marks rather than being the only distinction.
